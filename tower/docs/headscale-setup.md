# Headscale Setup Guide

## Overview

Tower runs [Headscale](https://github.com/juanfont/headscale) — a self-hosted implementation of the Tailscale control plane — with the built-in DERP relay enabled. Nodes (Unraid servers named `giraffe`, `cetriolo`, `ostrich`) run the standard Tailscale client pointed at this control server. The VPN subnet is `100.64.0.0/10`.

The Headscale container is defined in `tower/containers/headscale/docker-compose.yml` and managed by Ansible like all other stacks.

---

## Prerequisites

Before the first run, verify:

- **DNS resolves publicly.** Headscale requests a TLS certificate via ACME on startup. The DNS record for your tower hostname must be live and pointing at the tower IP before the container starts, otherwise ACME will fail and the container will crash-loop.
- **Ports are open on the firewall:**
  - `443/tcp` — HTTPS (control plane + HTTPS DERP)
  - `3478/udp` — STUN (used by DERP for NAT traversal)

## Pre-Deployment: Prepare Config Files Manually

The Headscale config is **not generated by Ansible** — you must create it on tower before running the playbook for the first time.

### 1. Create the directories

```bash
mkdir -p /opt/headscale/data
```

### 2. Create `/opt/headscale/config.yaml`

```yaml
server_url: "https://<HEADSCALE_SERVER_URL>"

listen_addr: "0.0.0.0:443"
metrics_listen_addr: "127.0.0.1:9090"
grpc_listen_addr: "127.0.0.1:50443"
grpc_allow_insecure: false

noise:
  private_key_path: /var/lib/headscale/noise_private.key

ip_prefixes:
  - "100.64.0.0/10"

derp:
  server:
    enabled: true
    region_id: 999
    region_code: "tower"
    region_name: "Tower (self-hosted)"
    stun_listen_addr: "0.0.0.0:3478"
  urls: []
  auto_update_enabled: false

database:
  type: sqlite
  sqlite:
    path: /var/lib/headscale/db.sqlite

tls_letsencrypt_hostname: "<HEADSCALE_SERVER_URL>"
tls_letsencrypt_cache_dir: /var/lib/headscale/cache
tls_letsencrypt_challenge_type: TLS-ALPN-01

dns_config:
  magic_dns: false
  nameservers: []

log:
  level: info
  format: text
```

Replace `<HEADSCALE_SERVER_URL>` with the public hostname of your tower. The `tls_letsencrypt_hostname` should be the bare hostname without `https://`.

## First-Run Steps

These steps are performed once after the container is running for the first time.

### 1. Retrieve the node key

After the container starts, get the node key that nodes will use to identify this server:

```bash
docker compose exec headscale headscale debug node-key
```

### 2. Create a user

Headscale requires at least one user (equivalent to a Tailscale account/tailnet). Create one for the homelab nodes:

```bash
docker compose exec headscale headscale users create homelab
```

### 3. Generate a preauth key

Create a reusable preauth key so nodes can register without manual approval:

```bash
docker compose exec headscale headscale preauthkeys create --user homelab --reusable --expiration 24h
```

Copy the key from the output. Share it securely with each node operator (or use it yourself on each Unraid host). After nodes are registered you can let the key expire or delete it.

## Key Commands Reference

| Action | Command |
|---|---|
| Create a user | `docker compose exec headscale headscale users create <name>` |
| List users | `docker compose exec headscale headscale users list` |
| Create a preauth key | `docker compose exec headscale headscale preauthkeys create --user <name> --reusable --expiration 24h` |
| List preauth keys | `docker compose exec headscale headscale preauthkeys list --user <name>` |
| List registered nodes | `docker compose exec headscale headscale nodes list` |
| Rename a node | `docker compose exec headscale headscale nodes rename --identifier <id> <new-name>` |
| Expire a node | `docker compose exec headscale headscale nodes expire --identifier <id>` |

All commands are run from the directory containing the Headscale `docker-compose.yml` (`/opt/containers/headscale/` on tower).
